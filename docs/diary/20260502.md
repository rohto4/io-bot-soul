## Docker常駐への定期投稿移行

GitHub Actionsのscheduleが安定しないため、定期投稿抽選をすべてローカルDocker常駐へ移した。

- [常駐アプリ本体](../../src/app.ts)
  - `start` に `postDrawIntervalMs` を追加し、pollingとは別にpost-draw timerを起動するようにした。
  - post-draw中に次のtickが来た場合は `postDraw.skip reason=already_running` を出す。
  - 起動直後にpost-drawは実行せず、最初の `POST_DRAW_INTERVAL_SECONDS` 経過後に抽選する。
- [起動入口](../../src/main.ts)
  - 起動ログへ `postDrawIntervalMs` を戻し、Docker常駐で投稿抽選も担当することを明確にした。
- [設定読み込み](../../src/config.ts)
  - `POST_DRAW_INTERVAL_SECONDS` を追加し、初期値を300秒にした。
- [環境変数テンプレート](../../.env.example)
  - `POST_DRAW_INTERVAL_SECONDS=300` を追加した。
- [GitHub Actions定期実行](../../.github/workflows/scheduled-post-draw.yml)
  - 定期投稿入口として使わないため削除した。
- [Dockerローカル常駐ガイド](../guide/docker-local-run.md)
  - 投稿抽選をDocker常駐で行う運用に更新した。
- [スクリプト概要](../guide/script-overview.md)
  - `npm start` がpollingとpost-drawの両方を担当する構成へ更新した。
- [投稿実行ルール](../spec/posting-runtime-rules.md)
  - 起動元をGitHub ActionsからDocker post-draw intervalへ変更した。
- [ユーザー作業](../imp/user-tasks.md)
  - `.env.local` とDockerログで確認する運用手順に更新した。
