## OpenCode / oh-my-openagent グローバル設定修正

opencodeとoh-my-openagentが全PJで動作しない問題を特定・修正した。

根本原因:
- グローバル `~/.config/opencode/opencode.json` にpluginもproviderも未設定
- `CHUTES_API_KEY` がWindowsユーザー環境変数に未登録（`.env.local` にしかなかった）

対応:
- `CHUTES_API_KEY` を `HKCU\Environment` に登録
- グローバル `opencode.json` を全面書き換え（plugin + Chutes全モデル + デフォルトmodel設定）
- `opencode debug config` で `plugin_origins.scope=global` を確認
- `docs/guide/opencode/oc-active-init.md` を2段階構成（全マシン一度 / PJごと）に改訂

## AI投稿プロンプト改善

投稿が「深夜のTL、〜。生活ログ〜。私えらいので〜」のワンパターンになっていた問題を改善。

原因: systemPromptの冒頭が「TLを観察し生活ログを記録・投稿」で固定されており、投稿例も全部同じ文脈だった。

対応:
- 冒頭を「Misskey高校・家・図書館・ラボ・商店街・ゲーセン・近所の河原を拠点に生活」に変更
- 多様性ルール追加（直前と同じ書き出し・締め方禁止、特定フレーズの3連続禁止）
- 直前3件の書き出し・締め方パターンをuserMessageに明示して回避指示
- 投稿例を全削除（ルールで十分）
- 重みづけ記憶参照: 直近7日全量・7〜30日3件おき・30〜60日10件おきのtiered SQL CTE
- 過去投稿への文脈フレーム「この蓄積と流れを踏まえた上でノートを生成してください」を追加
- キャラクター仕様を `character-spec.ts` に共通化
- AI API呼び出しを `chat-api.ts` に共通化（fallback含む）

## Phase 3: TL観測ノート・行動ガチャ・引用RN

5分tickの行動ガチャと、TL観測・引用RNの投稿フローを実装した。

### 行動ガチャ構造

```
5分tick
 └─ TL観測ガチャ（20%）当たり
     └─ 引用RNガチャ（20%）当たり → quote_renoteノート投稿  ← 全体4%（25回に1回）
     └─ 引用RNガチャ外れ or 失敗
         └─ TL観測テキストノート投稿                         ← 全体16%
 └─ TL観測ガチャ外れ
     └─ 通常ノート抽選（経過時間テーブル） or skip           ← 全体80%
```

### 実装ファイル

- `src/tl-scan.ts` — ホームTL取得・フィルタ（CW/renote/空テキスト除外）・`source_notes` 保存
- `src/ai/generate-tl-post.ts` — TL観測ベースの投稿生成プロンプト（個人特定なし・観察視点）
- `src/quote-pick.ts` — 許可済みユーザーから引用候補を選定
  - 1週間以内のノートのみ対象（古いノートはフォールバック）
  - 構造フィルタ後にAI安全判定（OK/NG）
- `src/ai/classify-quote-safety.ts` — 引用RN安全判定（医療・政治・個人情報等をNG、判定失敗はNG扱い）
- `src/ai/generate-quote-post.ts` — 引用に添えるコメント生成（1〜2文）
- `src/misskey/client.ts` — `getHomeTimeline` / `getUserNotes` / `createNote.renoteId` 追加

### 暫定値（m_runtime_settingで調整可）

| 設定キー | 暫定値 |
|---|---|
| `TL_OBSERVATION_POST_PROBABILITY` | 0.20 |
| `TL_OBSERVATION_NOTE_COUNT` | 20 |
| `TL_OBSERVATION_MIN_POSTS` | 3 |
| `QUOTE_RENOTE_PROBABILITY` | 0.20 |
