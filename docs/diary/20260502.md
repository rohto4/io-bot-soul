## Phase 3: TL観測ノート・行動ガチャ・引用RN

5分tickの行動ガチャと、TL観測・引用RNの投稿フローを実装した。

### 行動ガチャ構造

```
5分tick
 └─ TL観測ガチャ（20%）当たり
     └─ 引用RNガチャ（20%）当たり → quote_renoteノート投稿  ← 全体4%（25回に1回）
     └─ 引用RNガチャ外れ or 失敗
         └─ TL観測テキストノート投稿                         ← 全体16%
 └─ TL観測ガチャ外れ
     └─ 通常ノート抽選（経過時間テーブル） or skip           ← 全体80%
```

### 実装ファイル

- `src/tl-scan.ts` — ホームTL取得・フィルタ（CW/renote/空テキスト除外）・`source_notes` 保存
- `src/ai/generate-tl-post.ts` — TL観測ベースの投稿生成プロンプト（個人特定なし・観察視点）
- `src/quote-pick.ts` — 許可済みユーザーから引用候補を選定
  - 1週間以内のノートのみ対象（古いノートはフォールバック）
  - 構造フィルタ後にAI安全判定（OK/NG）
- `src/ai/classify-quote-safety.ts` — 引用RN安全判定（医療・政治・個人情報等をNG、判定失敗はNG扱い）
- `src/ai/generate-quote-post.ts` — 引用に添えるコメント生成（1〜2文）
- `src/misskey/client.ts` — `getHomeTimeline` / `getUserNotes` / `createNote.renoteId` 追加

## 行動ガチャの3フェーズ構造リファクタ

  - Phase 3（生成・投稿）: AI生成 + Misskey書き込み
- TL観測ガチャが当たった場合は通常ノートへ絶対に落ちない
- `too_few_summaries` / TL obs AI失敗 → skip（通常ノートへfall-through削除）
- `docs/spec/action-flow.md` にMermaidで全体フローを記録

## 記憶深度ガチャ（MemoryDepth）

AIが過去の投稿に引っ張られ過ぎる問題に対応するため、記憶の参照深度をロジック側で制御。

| depth | 確率 | 挙動 |
|---|---|---|
| `normal` | 90% | top3のみ（多様性制約用）。記憶セクションなし |
| `reminisce` | 5% | 60日内からランダム5件。「連想・発展させて」 |
| `reference` | 5% | 60日内からランダム1件。「このノートへの言及・続き」 |

- `note-hint.ts` に `MemoryDepth` 型と `drawMemoryDepth` を追加
- `generate-post.ts` でdepthに応じてDBフェッチ量とプロンプトを切り替え

## お題・口調・文体パターンガチャ（NoteHint拡張）

normal depthのノートの単調さ対策として3層のガチャを追加。

- **お題（20種）**: 場所・行動・感覚・思考の種
- **口調（6種）**: 独り言/考察・衒学的/自己肯定/発見・興奮/観察者/懐っこさ
- **文体パターン（4種、normal depthのみ）**:
  - 短文観察: 観察→感想の2〜3行
  - 思考の連鎖: 気づきが次の思考を呼ぶ3〜5行
  - 考察・断言型: 前提→論理→結論
  - 生活感・行動報告: 体験描写→感情→自己肯定
- 文体パターンは具体的な例文ではなく「行ごとの役割説明」で注入（内容の引っ張りを防ぐ）
- reminisce/referenceには文体パターン不要（記憶フレームが構成を誘導するため）
