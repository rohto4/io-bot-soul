## 「私えらいので」連続3回問題の修正

3件連続で同じ締め方になっていた原因を特定し、多層で対策した。

**原因の重なり:**
1. `noteStyles` の「生活感・行動報告」の `example` に「`「私えらいので〜」など`」と具体的に書かれていた
2. `採用要約` が「私えらいので〜フレーズを使う」と明記
3. 多様性ルールが「3回連続は禁止」でゆるい（2回まではOKと読める）
4. 直前パターンが先頭/末尾30文字だけで全体構成が見えない

**対策:**
- `src/note-hint.ts`: 「生活感・行動報告」の example から「私えらいので〜」を削除 → 「体験をまとめる自己肯定または軽い宣言（自分の言葉で）」に変更
- `src/ai/generate-post.ts`:
  - 多様性ルール: 「直近2件に含まれる場合は今回使わない」に強化
  - `buildUserMessage`: 直近2件の締め方に `watchPhrases`（「私えらいので」等）が含まれる場合、userMessage に今回の禁止フレーズとして明示
  - **直近パターン表示を全文表示に変更**: 書き出し/末尾30文字ではなく、直近2件の全文（200字まで）を表示。AIが行数・構成・フレーズを全部見た上で回避できるようにした

## 引用RN候補選定をホームTLベースに刷新

ランダムな許可ユーザー5人を選んで per-user API 呼び出しをする旧設計では、投稿数が少ないユーザーが繰り返し同じノートを引用される問題があった。

**新設計（`src/quote-pick.ts` 全面書き直し）:**
1. ホームTL 100件を取得（`QUOTE_RENOTE_TL_LIMIT` DB設定）
2. 許可済みユーザーのノートのみ抽出
3. 直近5回の引用RN済みユーザーを除外（`QUOTE_RENOTE_RECENT_USER_EXCLUDE` DB設定）
4. 引用済みノートID（`posts.quote_source_note_id`）を除外
5. 残った候補をシャッフルしてAI安全判定
- `getUserNotes` への per-user API 呼び出しを廃止、`getHomeTimeline` に一本化
- `quotePick.candidates` ログで候補数・除外数を可視化
