## 5. 判断

- ローカルAlpha実装には移行可能。
- 固定テンプレートの定期ノートに限れば、実投稿前ブロッカーは主に運用手順とActions設定。
- AI生成・分類、TL観測、体験候補、引用RenoteはP1以降でよい。
- AI設定値はGitHub variablesではなくDBマスタ管理へ寄せる。Actions secretsには `CHUTES_API_KEY` と `OPENAI_API_KEY` だけを置く。
- 投稿確率、最短投稿間隔、limit、rate limit、AI設定などの非secret運用値を `m_runtime_setting` に集約する方針へ変更した。
- エモーション画像添付はP1/P2で、初回実装を止める要件ではない。
- Chutesは使えるが、model idは `moonshotai/Kimi-K2.5-TEE` を指定する必要がある。
- OpenAI fallbackの `gpt-5.4-mini` は疎通済み。
- Chutes Kimiは `reasoning_tokens` を消費するため、短い分類でもtoken上限を小さくしすぎない。

## 10. セッション引継ぎ

- 日本語・UTF-8で作業する。
- secret、token、Cookie、未公開認証情報はリポジトリに書かない。
- `.env.local` には実キーが入っているため、表示・commitしない。
- Actionsに登録するAI keyは `CHUTES_API_KEY` と `OPENAI_API_KEY`。どちらもrepository secrets。
- AI provider、model id、timeout、retry、token上限、temperature、日次上限、fallback方針はDBマスタ管理にする。
- ChutesのKimi model idは `moonshotai/Kimi-K2.5-TEE`。`Kimi-K2.5-TEE` だけでは404になる。
- OpenAI `gpt-5.4-mini` は `max_tokens` ではなく `max_completion_tokens` を使う。
- Chutesは `max_tokens` を使う。
- Chutes Kimiは内部推論で `reasoning_tokens` を消費し、上限が小さいと `content = null` になる。
- `message.reasoning` / `message.reasoning_content` はログ本文や投稿に使わない。
- 定期ノートの初回実投稿前は、必ず `SCHEDULED_POSTING_ENABLED=false` のActions手動実行でskipを確認する。
- `SCHEDULED_POST_MIN_INTERVAL_MINUTES` はDBマスタ `m_runtime_setting` で `5` にする。30分以上空ける要件ではない。
- 初回投稿後は、直後再実行で `min_interval` skipを確認する。
