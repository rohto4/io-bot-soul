## action-flow-v2 実装をcommit/push

前セッションで実装済みだった変更（18ファイル・1539行追加・464行削除）をまとめてcommit。

### 変更の概要

行動ガチャの設計をPhase 3のv1フローからv2に刷新した。

**旧設計（v1）の問題点:**
- TL観測(20%)に入ってから引用RNガチャ(20%)を引く階層構造
- TL観測失敗時に通常ノートへのfallthrough
- tl_observation という独立した投稿 kind が存在し複雑

**新設計（v2）:**

```
引用RN（QUOTE_RENOTE_PROBABILITY: 20%）
  └─ 失敗/候補なし → skip（通常ノートに落ちない）

通常ノート（残り80%、最短間隔+確率テーブル）
  ├─ TL参照なし（TL_REFERENCE_PROBABILITY外れ: 40%）
  ├─ TL雰囲気言及（TL_REFERENCE × TL_VIBE_RATIO: 30%）
  └─ TL特定言及（TL_REFERENCE × TL_MENTION_RATIO: 10%）
```

- `posts.kind = 'tl_observation'` は廃止。すべて `kind = 'normal'`、詳細は `generated_reason` で区別
  - `no_tl` / `tl_vibe` / `tl_mention` / `quote_renote`
- 通常ノート生成時に `experience_logs` の体験記憶をプロンプトに弱注入（`EXPERIENCE_MEMORY_PROMPT_WEIGHT`）

### DB追加設定値（m_runtime_setting）

| 設定キー | 初期値 | 意味 |
|---|---|---|
| `QUOTE_RENOTE_PROBABILITY` | 0.20 | 引用RN独立ガチャ確率 |
| `TL_REFERENCE_PROBABILITY` | 0.50 | 通常ノートでのTL参照確率 |
| `TL_VIBE_RATIO` | 0.75 | TL参照内の雰囲気言及割合 |
| `TL_MENTION_RATIO` | 0.25 | TL参照内の特定言及割合 |
| `EXPERIENCE_MEMORY_ENABLED` | true | 体験メモリ注入の有効/無効 |
| `EXPERIENCE_MEMORY_SAMPLE_COUNT` | 50 | `experience_logs` からのサンプル数 |
| `EXPERIENCE_MEMORY_PROMPT_WEIGHT` | 50 | 影響度（0〜100） |
