# Release Readiness

## 判断

現時点の仕様は、ローカルPCでの限定運用・アルファ運用ならリリース可能。

ただし、misskey.ioで公開運用する前に、以下のゲートを満たす必要がある。

## 必須ゲート

### misskey.io運用ルール

- Botフラグを付ける。
- プロフィールまたはピン留めノートに、Bot管理者のmisskey.ioアカウントを記載する。
- ユーザー操作によりリプライや公開投稿を行う機能には、レートリミットを設ける。
- 不適切な単語には反応しないフィルタを設ける。
- 連続した公開投稿でタイムラインを埋めない。
- 投稿内容を定期的に人間が確認する。

### 投稿頻度

- 5分ごとの投稿抽選は許容する。
- ただし実投稿はrate limitを必ず通す。
- 初期公開運用では、1時間あたり最大5ノート、1日あたり最大50ノートを上限にする。
- 引用Renoteはさらに別枠で上限を設ける。
- 初期値として、引用Renoteは1日最大5回にする。
- リプライやフォロー起点のユーザー操作投稿が5分に5件を超えた場合、次の5分は休止してカウントをリセットする。
- 上限到達時は投稿をskip、または引用Renoteを抽象化投稿へfallbackする。

### 安全弁

- プロフィールにBotであること、参照の仕組み、停止方法を書く。
- ピン留めノートに同意条件、対象外、`/stop`、`/unfollow` を書く。
- 引用Renoteや体験候補に使う対象は、ピン留めノートへ❤した許可済みユーザーに限定する。
- TL観測は許可不要だが、個人名、引用Renote、元note文面の再利用を禁止する。
- CW、NSFW、個人情報、病気、事故、揉め事、政治、医療、投資、成人向け、攻撃的内容は採用しない。
- 削除済み・非公開化・参照不能noteは使わない。
- `/stop` と `/unfollow` は即時反映する。
- `/unfollow` はbot側から実際にフォロー解除し、ノート参照対象から除外する。

## リリース区分

### Alpha

目的: 自分のPCで動かし、投稿・DB・同意導線を検証する。

可能:

- 自分だけが監視できる範囲でのローカル常駐。
- 低頻度投稿。
- 手動で選んだテストユーザーのみ。

禁止:

- 大量フォロー。
- 大量引用Renote。
- 同意なしの体験候補化。

### Public MVP

目的: misskey.io上で通常運用を開始する。

開始条件:

- Botフラグ設定済み。
- 管理者アカウント明記済み。
- ピン留めノート公開済み。
- SQLite schema実装済み。
- rate limit実装済み。
- safety filter実装済み。
- `/stop` `/unfollow` 実装済み。
- 投稿ログとskip理由を確認できる。
